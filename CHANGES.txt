RS232_WEB_CLOUD_009_ButtonCommandMap
====================================

Цел: поправка на реалното поведение на бутоните/полета в cloud страницата.

Промени спрямо CLOUD_008:
- Премахнат е бутонът Print от cloud Main UI.
- Разрешени са допълнителни command queue команди:
  hold_start, hold_stop, hold_toggle,
  loop_start, loop_stop, loop_toggle,
  set_current_factor,
  set_measurement_options,
  set_phase_options.
- HOLD бутонът вече изпраща hold_start или hold_stop според текущия JSON state.
- LOOP бутонът вече изпраща loop_start или loop_stop според текущия JSON state.
- Target / Tol / Selection / Trigger by / AutoStop изпращат set_trigger.
- I factor / 3-phase voltage / L1 / L2 / L3 / Inductance изпращат set_measurement_options.
- Article промяна и Article Add изпращат set_article.
- Next No промяна, NO+ и NO- изпращат set_next_no.
- Export result и Clear table остават локални функции на browser таблицата.

Важно:
- Командите се записват само ако Render има:
  ALLOW_REMOTE_COMMANDS=1
  COMMAND_QUEUE_ENABLED=1
  COMMAND_TOKEN=...
- POST /api/request-command изисква COMMAND_TOKEN.
- ESP32 pull/ack изисква DEVICE_TOKEN.
