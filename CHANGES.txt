RS232_WEB_CLOUD_005_UI_StateMap
================================

Дата: 2026-06-27
Тип: cloud-only версия. RS232_WEB_108 firmware не е променян.

Основна цел:
- Фиксира връзката JSON поле → UI бутон/индикация.
- Разделя реалното DEVICE състояние от локалните функции на cloud страницата.
- Премахва подвеждащото поведение, при което Start/Stop Loop сменяше цвета си като локален auto-refresh.

Ново в UI:
- Добавен е отделен бутон: Cloud Auto Refresh.
- Start/Stop Loop вече следва само JSON полето loop_running.
- HOLD вече следва само JSON полето hold.
- Trigger status следва trigger_state / trigger_hit / trigger_value.
- VERIFY badge следва verified.
- Добавен е видим UI State Map панел.
- При натискане на DEVICE бутони се показва allow_remote_commands=0, без да се сменя състоянието им локално.

Ново в server.js:
- service name: RS232_WEB_CLOUD_005_UI_StateMap
- Добавен е GET /api/state-map.
- /health връща ui_state_map: v1.
- Command endpoints остават заключени.

Запазено:
- POST /api/push остава защитен с DEVICE_TOKEN.
- VIEW_TOKEN остава опционален за гледане.
- Няма tunnel към ESP32.
- Няма SD Tools, Settings, Diagnostics, FFT, директни команди към уреда.

Тестови payload файлове:
- sample_payload_basic.json
- sample_payload_full_state.json
- sample_payload_trigger_hit.json
- sample_payload_loop_running.json

Очаквани тестове:
- full_state: HOLD нормален, LOOP OFF, TRIG OK.
- trigger_hit: HOLD червен, TRIG HIT, LOOP OFF.
- loop_running: LOOP активен, HOLD нормален, TRIG ARMED.
